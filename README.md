# Uzumaki Portfolio: Fullstack Developer Portfolio & Admin Dashboard

A professional, feature-rich, fullstack developer portfolio designed with a sleek dark aesthetic, advanced interactive components, and a custom secure administration panel. This repository houses both the high-performance React client and the robust FastAPI backend.

---

## 🚀 Key Features

### 🌟 Interactive Client Portfolio
- **Fluid Visual Animations:** Built using `Framer Motion` for smooth interactive transitions, reveal-on-scroll animations, and hover micro-animations.
- **Dynamic Shinobi Stats:** Visual metrics displaying current status levels, including integration widgets that query Github, LeetCode, and Codeforces stats in real-time.
- **Responsive Photo Gallery:** A beautiful grid gallery with image lazy loading and full lightbox navigation for viewing high-resolution street/festival photography.
- **Markdown-Driven Scroll Blog:** A personal blogging platform rendering content directly from raw markdown using custom tailwind components.

### 🛡️ Secure Admin Panel (Hokage Office)
- **Role-Based Login Gate:** Admin credential validation using `bcrypt` password hashing and secure token-based session management.
- **Full CRUD Dashboards:** Dynamic UI to create, update, re-order, or delete:
  - **Blog Posts:** Autogenerated URL slugs, drafts/published statuses, and markdown content formatting.
  - **Photos catalog:** Camera bodies, lenses, aperture settings, categories, and custom display sorting.
  - **Skills Arsenal:** Mastery levels (Mastered / Learning), custom Simple Icons slugs, and visibility toggling.
  - **Credentials:** Secure form to change admin passwords directly from the dashboard.
- **Secret Encrypted Notes:** Write and store sensitive notes encrypted on the fly using **AES-256 GCM** encryption before database insertion.

## ⚡ Setup & Installation

### Prerequisites
- Node.js (v18+)
- Python (v3.10+)

### 1. Backend Configuration
Navigate into the `backend/` directory:
```bash
cd backend
```

Create and activate a Python virtual environment:
```bash
python -m venv .venv
# On Windows (PowerShell):
.\.venv\Scripts\Activate.ps1
# On macOS/Linux:
source .venv/bin/activate
```

Install Python dependencies:
```bash
pip install -r requirements.txt
```

Create a `.env` file in the `backend/` directory with the following variables:
```env
# Server Mode
DATABASE_URL=sqlite:///./portfolio.db # Defaults to local SQLite, or your Postgres URL

# Security Keys (Generate strong values for production)
JWT_SECRET=your_jwt_signing_key_here
NOTES_ENCRYPTION_KEY=your_32_byte_aes_key_here

# Cloudinary Integrations (For uploading portfolio photos)
VITE_CLOUDINARY_CLOUD_NAME=your_cloud_name
VITE_CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret

# Initial Admin Password (Used on first run seed)
ADMIN_INITIAL_PASSWORD=your_secure_password
```

Start the local backend server:
```bash
uvicorn main:app --reload --port 8000
```
The API documentation will be available at `http://localhost:8000/docs`.

### 2. Frontend Configuration
Return to the repository root directory:
```bash
cd ..
```

Install Node dependencies:
```bash
npm install
```

Create a `.env` file in the root directory:
```env
VITE_API_URL=http://localhost:8000
VITE_CLOUDINARY_CLOUD_NAME=your_cloud_name
VITE_CLOUDINARY_API_KEY=your_cloudinary_api_key
```

Run the development server:
```bash
npm run dev
```
The application will run locally at `http://localhost:5173`.

## 🗄️ Database Auto-Migrations
The application uses SQLAlchemy to automatically manage database structures. 
On backend startup:
1. All missing tables (`admin`, `blog_posts`, `photos`, `notes`, `skills`) are auto-created.
2. The initial Admin account is seeded using the password set in `ADMIN_INITIAL_PASSWORD`.
3. In-app migration logic automatically inspects tables and adds newer columns (such as the `category` column in `photos` and `is_visible` column in `skills`) to prevent schema conflicts on production environments (e.g. Vercel/Render/Railway) without manual SQL commands.

## 🔒 Security Best Practices Implemented
- **Restricted CORS Domains:** Requests originating from cross-origin addresses are validated against a strict domain whitelist, specifically restricting Vercel branch/production endpoints of the project owner.
- **HTTP-Only Cookies:** JWT authentication tokens are transmitted inside secure, HTTP-only, and samesite cookies to mitigate Cross-Site Scripting (XSS) session stealing.
- **Rate-Limiting:** Sensitive entry points (login, password updates) are wrapped with rate-limiters preventing brute-force dictionary attacks.
- **Hardware-backed Cryptography:** AES-256-GCM encryption ensures secret notes remain completely unreadable if database contents are leaked.
- **Safe Uploads:** Server-side upload mechanisms enforce UUID renaming and whitelisted file extension checks to mitigate path traversal and code injection attacks.
